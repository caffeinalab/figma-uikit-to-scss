Nodes selectors

The following settings allow the plugin to retrieve the right nodes to be exported as scss variables. There are three types of groups:

  • Palette: identify a color variable
  • Typography: identify a text style
  • Components: identify for example a button or an input component

Based on this configuration, the plugin will search nodes with name like @uikit-typo-h1-mobile, i.e. accorging the following format:
{BASE_PREFIX}{SEPARATOR}{GROUP_PREFIX}{SEPARATOR}{variable_name}{SEPARATOR}{mobile/tablet/desktop}

Base prefix (to identify a uikit element)
Palette group prefix

The plugin will export the first fill color of the node

Typography group prefix

The plugin will export `font-family`, `font-size`, `font-style`, `font-weight`, `line-height`, `letter-spacing`, `text-transform`, `color` of the node

Components group type

The plugin supports the Figma's variants too. Check the related option if you use them in your components design workflow.

Components group prefix

The plugin will export `opacity`, `padding`, `border`, `border-radius`, `background-color` of the node and the `color`(first fill color) of the first visible children

Child reference prefix

This selector will be used in components to find the child node that you want to use as styles source. It is useful for example when the node declared as a component is styleless (e.g. a node group).
- @uikit-component-button
-- @uikit-ref

Prefixes separator

The separator used to concatenate nodes selectors (e.g. @uikit-typo-h1-mobile)

Scss maps names

Choose the variable names (without $) used by the exporter to generate the uikit sass file.
e.g. if you choose 'color' as palette variable the plugin will export `$colors: ( "white": "#FFFFFF", ... )`
